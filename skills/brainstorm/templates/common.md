# 通用追问技巧

> **适用于**: 所有模式的补充提问

---

## 当用户回答模糊时

```yaml
clarification_prompts:
  - "能举个例子吗？"
  - "具体是指？"
  - "你说的 {术语} 可以详细解释一下吗？"
  - "有没有具体的场景可以说明？"
```

---

## 当对话停滞时

```yaml
conversation_prompts:
  - "还有其他需要考虑的吗？"
  - "这个方案有什么潜在问题？"
  - "如果是你，你会怎么选？"
  - "我们漏掉了什么吗？"
```

---

## 当需要深入时

```yaml
deepening_prompts:
  - "为什么你觉得 {方案} 更好？"
  - "{方案} 有什么劣势吗？"
  - "如果我们放宽 {约束}，选择会变化吗？"
  - "从 {另一个角度} 来看，有什么不同？"
```

---

## 当准备收敛时

```yaml
convergence_prompts:
  - "综合来看，哪个方案最合适？"
  - "有最佳选择吗？"
  - "我们可以做决定了吗？"
  - "还有什么顾虑吗？"
```

---

## 多选题技巧

当可能时，使用多选题而非开放性问题：

```yaml
示例:
  ❌ "你想要什么功能？"
  ✅ "你更关注哪个功能？
      a) 智能推荐
      b) 搜索优化
      c) 个性化设置
      d) 其他（请说明）"

优势:
  - 更容易回答
  - 方案明确
  - 便于收敛
```

---

## 一次一问原则

每次只问一个问题，避免信息过载：

```yaml
❌ "请描述你的用户群体、他们的技术水平、使用场景和期望功能"
✅ "谁会使用这个功能？"
   → "大概有多少用户？"
   → "用户的熟练度如何？"
```

---

## YAGNI 原则

持续应用 "You Aren't Gonna Need It"：

```yaml
过度设计:
  - "要不要加个 AI 推荐？"
  - "支持多租户吗？"
  - "能对接第三方吗？"

YAGNI 反问:
  - "这是核心功能必需的吗？"
  - "能否后续迭代再添加？"
  - "没有它会影响 MVP 吗？"
```

---

## 总结确认技巧

在每个阶段结束时进行确认：

```yaml
CLARIFY 结束:
  "所以，我们明确的关键术语是：{列表}，对吗？"

EXPLORE 结束:
  "我们考虑了这些方案：{列表}，还有其他吗？"

CONVERGE 结束:
  "综合来看，我们倾向于 {方案}，因为 {理由}，对吗？"
```
