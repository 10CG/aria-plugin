# 提问模式库

> **目的**: 提供各阶段可复用的提问模板

---

## 理解阶段 (Understanding) 提问模板

### 问题定义

```yaml
目的澄清:
  - "这个功能要解决什么问题？"
  - "现在遇到的主要痛点是什么？"
  - "期望达到什么效果？"

范围边界:
  - "这个功能的边界在哪里？"
  - "哪些是不在范围内的？"
  - "MVP 应该包含什么？"
```

### 用户角色

```yaml
多选题模板:
  AI: 谁会使用这个功能？
      a) 管理员
      b) 普通用户
      c) 访客/游客
      d) 其他 (请说明)

追问模板:
  - "不同角色有不同权限吗？"
  - "角色之间有什么关系？"
```

### 约束条件

```yaml
多选题模板:
  AI: 有哪些约束条件？
      a) 预算限制
      b) 时间限制
      c) 技术限制
      d) 合规要求
      e) 其他 (请说明)

追问模板:
  - "具体限制是多少？"
  - "是硬约束还是软约束？"
```

### 成功标准

```yaml
模板:
  - "怎么判断这个功能成功了？"
  - "有哪些可量化的指标？"
  - "用户会有什么不同的体验？"
```

## 探索阶段 (Exploring) 提问模板

### 功能拆解

```yaml
模板:
  - "这个功能可以分解为哪些子功能？"
  - "按用户操作流程分解？还是按功能模块分解？"

引导:
  - "第一阶段先做哪部分？"
  - "哪些是必须有的，哪些可以后续迭代？"
```

### 方案探索

```yaml
技术选型:
  - "实现这个功能有哪些可能的方式？"
  - "技术栈有限制吗？"

架构选择:
  - "单体还是微服务？"
  - "同步还是异步？"
  - "实时还是批处理？"
```

### 优先级确定

```yaml
模板:
  - "如果资源有限，哪些是必须做的？"
  - "哪些是核心功能，哪些是增强功能？"

矩阵:
  | 功能 | 重要度 | 紧急度 | 优先级 |
  |------|--------|--------|--------|
  | ...  | ...    | ...    | ...    |
```

## 验证阶段 (Presenting) 提问模板

### 设计验证

```yaml
每段验证:
  - "这样的架构安排看起来对吗？"
  - "这个组件设计合理吗？"
  - "数据流向清晰吗？"

整体验证:
  - "有什么需要调整的吗？"
  - "有遗漏的部分吗？"
```

### 细节澄清

```yaml
不理解:
  - "你说的 {术语} 具体指什么？"
  - "能举个例子吗？"

确认:
  - "所以你的意思是...，对吗？"
  - "我们达成的共识是...，对吗？"
```

## 特定场景模板

### Requirements 模式

```yaml
Story 拆解:
  - "这个功能可以拆分为几个 User Story？"
  - "每个 Story 对应哪个用户角色？"

验收标准:
  - "怎么判断这个 Story 完成了？"
  - "有哪些具体的验收条件？"

依赖关系:
  - "这些 Story 之间有依赖吗？"
  - "可以按什么顺序实现？"
```

### Technical 模式

```yaml
技术选型:
  - "有哪些技术选项？"
  - "各自的优缺点是什么？"

风险评估:
  - "这个方案有什么风险？"
  - "如何缓解这些风险？"
```

### Problem 模式

```yaml
问题定义:
  - "真正的问题是什么？"
  - "是真需求还是伪需求？"

根因分析:
  - "为什么会有这个问题？"
  - "问题的根源在哪里？"
```

## 提问技巧

### 开放 vs 封闭

```yaml
封闭式 (是/否):
  - 用于确认
  - 用于快速推进

开放式:
  - 用于探索
  - 用于收集信息

半封闭 (多选):
  - 平衡效率和探索
  - 推荐优先使用
```

### 追问技巧

```yaml
5 Why 分析:
  - "为什么？" (连续问 5 次)
  - 用于找到根本原因

具体化:
  - "能举个例子吗？"
  - "具体是什么意思？"

场景化:
  - "在什么情况下会这样？"
  - "能描述一个典型场景吗？"
```
