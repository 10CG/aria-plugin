# Problem 模式完整对话示例

> **场景**: 用户想给应用加个 AI 功能，但不清楚具体需求

---

## 对话记录

```yaml
用户: /brainstorm problem "我想给应用加个AI功能"

AI: 让我们先理解一下你想要解决的问题。

     [问题澄清]
     1. 你说的 AI 功能具体指什么？
      - a) 智能推荐
      - b) 自动补全
      - c) 对话助手
      - d) 其他

用户: c，我想做一个智能客服

AI: 明白了，智能客服助手。让我们继续澄清：

     2. 这个客服要解决什么问题？
      - 当前有什么痛点？
      - 期望达到什么效果？

用户: 现在客服经常不在，用户问题得不到及时回答。
     我希望用户能随时得到帮助。

AI: [记录问题核心: 及时性]

     3. 关于回答质量:
      - 需要多"智能"？
      - 简单 FAQ 自动回答够吗？
      - 还是需要理解复杂问题？

用户: 至少能回答产品相关的问题，复杂问题再转人工。

AI: [记录约束: 产品知识库 + 人工兜底]

     4. 有什么限制条件吗？
      - 预算范围？
      - 时间要求？
      - 合规要求？

用户: 预算有限，最好每月不超过500块。
     数据不能出私有机房，有合规要求。

AI: [记录约束: 预算<$500 + 私有化部署]

     让我总结一下我们的讨论:

     ╔═══════════════════════════════════════════════════════════╗
     ║                     问题定义                               ║
     ╠═══════════════════════════════════════════════════════════╣
     ║ 核心问题: 用户咨询得不到及时响应                           ║
     ║ 目标用户: 产品使用用户                                     ║
     ║ 成功标准: 24/7 可用，产品问题自动回答率 > 80%              ║
     ╠═══════════════════════════════════════════════════════════╣
     ║ 约束条件:                                                  ║
     ║ - 预算: < $500/月                                         ║
     ║ - 部署: 私有化，数据不出境                                 ║
     ║ - 功能: 产品知识库问答，复杂问题转人工                     ║
     ╚═══════════════════════════════════════════════════════════╝

     建议: 创建 PRD 来详细规划这个功能。

     下一步:
     [1] 继续头脑风暴，分解 User Stories
     [2] 创建 PRD 文档
     [3] 记录决策，稍后继续

用户: 1

AI: [切换到 requirements 模式]
     让我们把智能客服分解成具体的 User Stories...
```

---

## 关键点分析

| 阶段 | 目标 | 技巧 |
|------|------|------|
| 开场 | 理解用户意图 | 提供多选项，快速定位 |
| 澄清 | 核心问题 | 痛点导向，挖掘深层需求 |
| 探索 | 约束收集 | 分类提问（预算/时间/合规） |
| 收敛 | 问题定义 | 结构化总结，边界确认 |
